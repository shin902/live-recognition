# 機能提案: リアルタイム音声入力とVAD（Groq連携版）

## 背景
本アプリケーションは低レイテンシの音声入力を目指しています。当初はElevenLabsを予定していましたが、速度とコスト効率の観点から **Groq (Whisper Large v3)** を採用することにしました。
これを実現するために、まずはマイクから音声を取得し、ブラウザ側（ローカル）で発話区間を検出（VAD）して、APIに送信可能な音声セグメントを切り出す仕組みが必要です。

## ゴール
レンダラープロセスに発話区間検出（VAD）を実装し、Groq APIでの文字起こしに向けた音声セグメントのキャプチャと整形を可能にします。

## スコープ
- **マイクアクセス**: Electronのレンダラープロセスでのマイク使用許可の取得とハンドリング。
- **VADの統合**: `@ricky0123/vad-web` を使用した、話し始めと話し終わりの検知。
- **音声バッファリング**: 発話中の生音声（Float32Array）のキャプチャ。
- **音声変換**: 生音声をGroq APIが受け入れ可能なWAV形式に変換。
- **UIフィードバック**: 現在の状態（待機中、聞き取り中、発話検知、処理中）の視覚的表示。

## スコープ外
- **API連携**: このフェーズではGroq APIへの実際の送信は行わず、モックまたはコンソールログに留めます。
- **テキスト注入**: 他のアプリへのテキスト自動入力。
