# Proposal: add-floating-window

## Summary

Voice Inkのような**常に最前面に表示されるフローティングウィンドウ**をElectronアプリに実装する。
画面下部に固定された細い横長バーとして表示され、macOSネイティブアプリのような洗練されたUIを提供する。

## Motivation

現在のLive Recognitionアプリは、標準的なElectronウィンドウとして動作している。
音声認識の結果をリアルタイムで確認しながら他のアプリケーション（ブラウザ、エディタ等）を操作するユースケースでは、
認識結果を表示するウィンドウが常に視認できる状態であることが重要。

フローティングウィンドウにより：
- 他のアプリの上に常に表示されるため、音声認識結果を常時確認可能
- コンパクトなUIにより、画面の大部分を他のアプリで使用可能
- 高度に洗練されたデザインでプレミアム感を演出

## Scope

### In Scope

- **フレームレス透明ウィンドウ**: ウィンドウ枠を削除し、背景を透明化
- **常に最前面表示**: `alwaysOnTop: true` で他のウィンドウより上に表示
- **画面下部固定配置**: 画面下部中央に配置、サイズは固定（リサイズ不可）
- **CSSスタイリング**: 半透明黒 + blur効果で角丸のバーデザイン
- **Dock非表示**: macOSのDockにアイコンを表示しない

### Out of Scope（将来実装）

- **フルスクリーンアプリ対応**: 他アプリのフルスクリーン上に表示する機能（`visibleOnFullScreen`）
- **位置のドラッグ移動**: ユーザーによるウィンドウ位置の変更

## Dependencies

- 既存の `BrowserWindow` 設定（`src/main/index.ts`）の大幅な変更
- CSS スタイルシート（`src/renderer/App.css`）の更新

## Risks & Mitigations

| リスク | 緩和策 |
|--------|--------|
| macOS以外での動作不確実性 | project.mdで「macOS専用」と明記されているため問題なし |
| `transparent: true` 時のパフォーマンス | 最小限のウィンドウサイズにより影響を抑える |
| CSS blur効果の描画負荷 | 控えめなblur値（10px程度）を使用 |

## Success Criteria

1. アプリ起動時に画面下部中央にフローティングバーが表示される
2. ウィンドウが常に他のアプリより上に表示される
3. ウィンドウ枠がなく、背景が透明で、角丸の黒半透明バーが表示される
4. Dockにアイコンが表示されない
5. ウィンドウのサイズ変更・移動ができない
